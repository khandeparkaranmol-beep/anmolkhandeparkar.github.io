The Pipeline

A sequential 5-stage pipeline where each stage feeds directly into the next. The whole thing runs end-to-end in under 2 minutes.

01
Website Extraction
Jina AI Reader converts any company URL — including React SPAs and Angular apps — into clean, parseable markdown.
Jina AI
02
Structured Extraction
Claude API with tool calling + schema enforcement extracts sector, revenue, size, HQ into guaranteed structured JSON.
Claude API
03
Rule-Based Filter
Hard filters eliminate mismatched funds. Cuts 500+ funds down to ~15 candidates fast and cheap.
Rule Engine
04
LLM Ranking
Claude reasons across candidates — scoring each on thesis fit, portfolio pattern, deal size alignment.
Claude API
05
Ranked Output
3–5 PE funds, each with plain-English reasoning. Explainable, not just a list.
Output
Why Hybrid — Not Pure LLM
Asking an LLM to reason across 500 funds at once is expensive, slow, and drifts. Hard rules eliminate the obviously wrong candidates first — then LLM reasoning handles the nuanced fit scoring. Rules for speed. LLM for judgment.
Why Tool Calling for Extraction
Free-form extraction hallucinates missing fields and returns inconsistent shapes. Schema enforcement via tool calling guarantees every company profile has the same structure. Deterministic inputs → reliable outputs.
